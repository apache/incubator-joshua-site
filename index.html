Joshua Decoder

Joshua is an open-source statistical machine translation decoder for hierarchical and syntax-based machine translation, written in Java. It is developed as a research collaboration between the Human Language Technology Center of Excellence at Johns Hopkins University and Chris Callison-Burch's research group at the University of Pennsylvania.

The latest version of Joshua is version 5.0. Its changes are described in the paper

Joshua 5.0: Sparser, better, faster, server BibTeX
Matt Post, Juri Ganitkevitch, Luke Orland, Jonathan Weese, Yuan Cao, and Chris Callison-Burch
WMT 2013
You might also be interested in older papers describing Joshua 1.0, the Joshua demo paper, 2.0, 3.0, and 4.0.

Features

The main features of Joshua 5.0 are as follows:

  • It's written in Java, so it's fast and uses minimal memory
  • Decoding
    • Translates with arbitrary SCFGs, with specific support for Hiero, SAMT, and GHKM
    • Translation of lattices (e.g., for speech or word segmentation) in PLF format
    • Support for tens of thousands of parameters with an underlying sparse feature representation and access to large-scale discriminative tuners like PRO and batch MIRA
    • Left-state minimization of LM state via KenLM
    • "Forced" decoding to find all derivations between a source and target sentence
  • Contains a script that handles the complete MT pipeline, from data preparation to alignment, model training, decoding, evaluation, and analysis.
  • Extraction of very large-scale Hiero and SAMT grammars with Thrax, our Hadoop-based grammar extractor

Usage

Check out our large collection of user documentation for information on downloading and setup, a quick start guide, detailed information on different Joshua components, common problems, and more.

The easiest way to use Joshua is to use the pipeline script which is included with the source code. This script supports multiple use cases; see the documentation for more information. For more detail, Chris Callison-Burch describes how to manually run the steps of the machine translation pipeline. (This information is a bit outdated and much of it is automated by the pipeline, but the broad steps still apply).

If you run into difficulty, feel free to email the Joshua Technical Support Group, or search its archives.

Other decoders

There are many other publicly-available open-source decoders available. Some of them are:

  • cdec, an SCFG decoder
  • Moses, the defacto phrase-based translation system which also supports hierarchical decoding
  • Jane a hierarchical phrase-based system
  • Phrasal, Stanford's phrase-based system
